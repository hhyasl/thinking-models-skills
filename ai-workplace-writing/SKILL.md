# AI职场写作

这是从《AI职场写作：无须再记提示词，人机共创好文章》（黎志锋、杨琳）一书中提炼的AI职场写作方法论体系，帮助用户从"指令输入"到"创意共创"全面升级写作能力。

## 适用场景

- 用户需要写职场文档（邮件、周报、总结、请示、报告、竞聘材料等）
- 用户觉得AI写的东西太"AI味"、太模板化
- 用户面对复杂写作任务不知如何拆解
- 用户需要提升提示词质量，让AI输出更精准
- 用户需要搭建文章结构或优化逻辑层次

## 一、主流程：AI写作六步法

遵循职场写作黄金法则——"自上而下地表达"，将写作拆解为六大环节，环环相扣、递进推进。

### 六步法总览

| 步骤 | 名称 | 核心任务 | 典型提问 |
|------|------|---------|---------|
| ① | **定向** | 明晰主旨，少走弯路 | "这篇文章的核心目的是什么？" |
| ② | **架构** | 搭建结构，纲举目张 | "帮我用LCN结构法给几个方案" |
| ③ | **扩展** | 强化论点，丰富内容 | "这个论点需要哪些论据支撑？" |
| ④ | **起草** | 人机共创，快速成稿 | AI写初稿 → 人类注入灵魂和风格 |
| ⑤ | **润色** | 回归个性，增强特色 | "让语言更像我的风格" |
| ⑥ | **定制** | 调整风格，适配场景 | "针对老板/客户/同事分别改写" |

**使用提示：** 六步法可**按需跳用**。简单任务从第④步或第⑤步开始即可，复杂任务建议从第①步开始完整走一遍。

## 二、提示词层

### 2.1 RCOR指令框架（基础提示词框架）

| 要素 | 含义 | 示例 |
|------|------|------|
| **R** Role | AI扮演的角色身份 | "你是资深管理顾问" |
| **C** Context | 背景信息 | "公司正在推进数字化转型" |
| **O** Objective | 目标/写作目的 | "说服管理层采纳新方案" |
| **R** Requirement | 具体输出要求 | "800字以内，分三段，附数据" |

**完整提示词模板：**
> 你是[角色]。请撰写[文档类型]，用于[场景/读者]。背景是[相关信息]。目标/主旨是[核心观点]。要求：[格式、篇幅、风格等约束]。

**典型提问：** "请基于RCOR框架帮我设计一个写竞聘材料的提示词，角色是资深HR顾问。"

### 2.2 温度参数（输出调性控制）

| 温度 | 特征 | 适用场景 |
|------|------|---------|
| 0.2~0.3 | 保守、稳妥、一致性强 | 财务报告、合规文档、制度文件 |
| 0.5~0.6 | 平衡 | 日常工作邮件、汇报、方案 |
| 0.7~0.9 | 有创意、有想象力 | 广告文案、品牌故事、创意写作 |

**典型提问：** "帮我写一份员工激励信，温度参数0.7，用数据和故事结合，语气温暖。"

### 2.3 四种写作指令

| 类型 | 适用场景 | 操作方法 |
|------|---------|---------|
| **直接指令** | 任务明确、简单直接 | 一次性给全信息，让AI交付 |
| **链式指令** | 多步骤复杂任务 | 分步骤引导 → 每一步确认后再继续 |
| **思维链指令** | 需要推理的分析任务 | "让我们一步步思考" / "请输出推理过程" |
| **火力侦察法** | 探索方向、拓展创意 | "先给5个切入角度，我再选方向" |

## 三、提问法层

### 3.1 布鲁姆提问法（六层递进）

借鉴布鲁姆认知分类法，从低阶到高阶系统化提问：

```
低阶 →  记忆 → 理解 → 应用 → 分析 → 评价 → 创造  ← 高阶
```

| 层级 | 提问示例 |
|------|---------|
| 📖 记忆 | "这个概念的定义是什么？" |
| 🧠 理解 | "用比喻解释这个概念" |
| 🛠️ 应用 | "这个概念在实际工作中怎么用？" |
| 🔍 分析 | "这个方案的优缺点是什么？" |
| ⚖️ 评价 | "你认为哪种方案更好？为什么？" |
| 💡 创造 | "基于这些信息，提出一个新方案" |

**典型提问：** "请用布鲁姆六层递进帮我深度分析这个主题，从记忆到创造每个层级各提一个问题。"

### 3.2 反向提问法（发现盲点）

让AI扮演不同角色来追问你，帮你发现认知盲点：

- **步骤：** 设定角色身份 → 明确关注重点 → 限定问题数量（2~3个）
- **典型角色：** 投资人、用户、法务顾问、上级领导、质疑者

**典型提问：** "假设你是投资人，从ROI和退出机制角度，对这份商业计划书提出三个问题。"

### 3.3 元问题提问法（提升问题质量）

当提问方向不确定时，先质疑问题本身：

1. **质疑问题本身**：当前问题是否切中要害？
2. **重构提问框架**：我为什么这么问？背后真正的目标是什么？
3. **切换提问视角**：从不同利益相关方重新设问
4. **追问问题前提**：这个问题的前提假设是否成立？

**典型提问：** "我需要写一篇推文但思路很乱，请用元问题提问法帮我找到真正该问的问题。"

## 四、结构设计层

### 4.1 LCN结构法（文章结构三要素）

| 要素 | 含义 | 典型工具 | 权重调节 |
|------|------|---------|---------|
| **L** 逻辑 Logic | 信息内在关联，严谨一致 | 金字塔原理、因果链 | 逻辑权重↑ → 理性论证 |
| **C** 认知 Cognition | 降低理解成本 | 降维解释、层级引导 | 认知权重↑ → 深入浅出 |
| **N** 叙事 Narrative | 创造阅读体验 | SCQA模型、峰终定律 | 叙事权重↑ → 故事驱动 |

**使用方式：** 告诉AI你想要的权重比例，让AI自动生成对应方案。例如："逻辑40%+叙事40%+认知20%"

**典型提问：** "请用LCN结构法帮我设计这篇文章的两个结构方案，一个逻辑偏重，一个叙事偏重，分别给出目录大纲。"

### 4.2 MECE法则（结构化思维核心）

**Mutually Exclusive, Collectively Exhaustive** — 相互独立、完全穷尽

**三大框架：**
| 框架 | 说明 | 示例 |
|------|------|------|
| 要素法 | 拆解为核心组成部分 | 马斯洛需求层次、波特五力 |
| 流程法 | 按时间/逻辑顺序拆解 | PDCA循环、六步法本身 |
| 矩阵法 | 交叉分析不同变量 | SWOT、BCG矩阵 |

**典型提问：** "请用MECE法则帮我分类这个方案的优缺点，确保分类不重叠、不遗漏。"

### 4.3 SCQA模型（叙事结构）

```
情境（Situation）→ 冲突（Complication）→ 问题（Question）→ 答案（Answer）
```

**典型提问：** "请用SCQA模型帮我写一份项目启动汇报的开头部分。"

### 4.4 黄金圈法则

```
Why（为什么做）→ How（怎么做）→ What（做什么）
```

**典型提问：** "请用黄金圈法则帮我组织这份方案的开篇，先讲为什么再讲怎么做。"

## 五、主旨层

### 5.1 主旨3C标准

| 标准 | 含义 | 检查问题 |
|------|------|---------|
| **Clear** 清晰 | 读者一听就懂 | "一句话说清楚核心观点了吗？" |
| **Concise** 简明 | 无冗余信息 | "能不能再删一半字数？" |
| **Compelling** 有冲击力 | 抓住注意力 | "读者看完会行动吗？" |

### 5.2 主旨五种形式
- **观点型**：亮明立场和判断
- **问题型**：围绕一个核心问题展开
- **行动型**：号召读者采取行动
- **价值型**：强调能给读者带来什么
- **概念型**：解释一个新概念

**典型提问：** "我想写一篇关于远程办公效率的文章，请用3C标准帮我检验主旨，并推荐最适合的主旨形式。"

## 模型组合使用建议

| 场景 | 组合链路 |
|------|---------|
| 写一封重要的工作邮件 | RCOR框架（提示词）→ 直接指令（一步完成）→ 温度参数0.3（正式） |
| 写一份竞聘材料 | 六步法完整（第①~⑥步）→ SCQA（开篇）→ 黄金圈（主体结构） |
| 分析一个复杂业务问题 | 元问题提问法（找对方向）→ 布鲁姆提问（深度分析）→ MECE（梳理结构） |
| 快速出稿一篇公众号文章 | 火力侦察法（选选题）→ LCN结构法（定框架）→ 六步法④起草+⑤润色+⑥定制 |
| AI写作质量不佳 | 诊断：RCOR是否完整？→ 温度参数是否合适？→ 是否需要链式指令分步引导？ |

## 提醒

- **"用魔法打败魔法"**：忘记RCOR框架时，直接让AI帮你设计提示词
- **"火力侦察法"**是最万能的破局方式：先让AI发散再你自己收敛
- **温度参数**是调节AI输出的"秘钥"，习惯在指令中加入"温度参数约0.X"
- AI写作六步法可以根据任务复杂度**灵活跳用**，不必每次都走完六步
- 核心原则：**AI负责"骨架与填充"，人类把控"灵魂与价值观"**
