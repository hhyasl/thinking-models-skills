# 通用思维模型

这是从《AI时代的个人成长·投资·财富自由》一书中提炼的一组通用思考框架，覆盖从"模糊问题"到"清晰结论"的完整思维链路。

## 适用场景

- 用户需要分析一个复杂、模糊的问题
- 用户需要写方案、报告或构建知识体系
- 用户不知道如何向AI有效地提问
- 用户的信息杂乱无章，需要理清结构
- 用户需要评估不同选项或做比较分析

## 核心模型

### 1️⃣ 概念化思维

将模糊、抽象的概念**精确定义并划定边界**，使其成为可讨论、可推理的思考单元。

**操作流程：**
1. 识别模糊词（如"风险""价值""效率""质量"）
2. 追问定义边界：这个词具体指什么？包含什么、不包含什么？
3. 用AI对比不同定义：让AI列出该概念的主流定义差异，选定最贴合当前场景的一个
4. 生成个人化工作定义并记录

**典型提问：** "帮我澄清'风险'这个概念，列出几种主流的定义框架，并说明它们的适用场景差异。"

### 2️⃣ 结构化思维

将碎片化信息按**逻辑关系组织为框架**，遵循MECE原则（Mutually Exclusive, Collectively Exhaustive —— 相互独立、完全穷尽）。

**操作流程：**
1. 收集所有相关信息（可用脑暴或AI发散）
2. 寻找分类维度，确保同层之间不重叠、不遗漏
3. 用思维导图或大纲呈现结构
4. 验证：每一项是否真的独立？是否真的穷尽？

**典型提问：** "帮我用MECE原则对影响房价的因素进行分类，确保各维度独立且完整。"

### 3️⃣ 系统化思维

超越静态分类，看到**要素之间的动态因果关系**。

- **区分"静态框架"与"动态演化"** —— 一个是对现状的分类（结构化），一个是对变化的建模（系统化）
- 识别**反馈回路**：正反馈（加速/放大）和负反馈（平衡/稳定）
- 识别**延迟效应**：原因和结果在时间上可能分离

**典型提问：** "帮我画一个系统因果回路图，分析'房价上涨→居民购房意愿'之间的正负反馈关系。"

### 4️⃣ 金字塔原理（麦肯锡）

一种以结论为导向的纵向逻辑结构。

**四大原则：**
1. **结论先行**：先给出核心观点
2. **以上统下**：上层论点统摄下层论据
3. **归类分组**：MECE原则组织同层信息
4. **逻辑递进**：同层论点按时间/结构/重要性排序

**AI角色设定：** 让AI扮演"结构化编辑"或"逻辑审计师"，检查你的论述是否符合金字塔结构。

**典型提问：** "以下是我想表达的观点，请帮我用金字塔原理重新组织，确保结论先行、以上统下。"

### 5️⃣ 3W1H提问模型

把模糊问题转化为AI可推理的结构化问题。

| 维度 | 含义 | 示例 |
|------|------|------|
| **What** | 具体面临什么问题？对象、范围？ | "我在分析新能源行业的投资机会" |
| **Why** | 为什么解决？动机、约束、权衡？ | "我有一笔闲钱想长期配置，能接受中等风险" |
| **When/Where** | 场景与时间窗口？现实边界？ | "投资期限5年以上，不考虑短期波动" |
| **How** | 希望AI产出什么？方案/步骤/对比？ | "请帮我列出3种入场策略，各有何优劣" |

**核心心法：**
- 问题越具体，AI的回答越有价值
- 始终提供上下文和约束条件
- 明确告诉AI你期望的输出格式

**典型提问：** "（What）我在做职业转型决策（Why）有家庭负担，不能冒险裸辞（When/Where）现在市场环境一般（How）请帮我做一个风险评估矩阵，对比两种方案的优劣势。"

## 模型组合使用建议

| 场景 | 组合链路 |
|------|---------|
| 从零分析一个复杂问题 | 概念化 → 结构化(MECE) → 系统化(因果) |
| 写一份方案报告 | 3W1H提问(澄清需求) → 金字塔原理(组织输出) |
| 向AI高质量提问 | 概念化(澄清关键词) → 3W1H(构建提问) → 金字塔(验收答案) |
| 做竞品/行业对比分析 | 结构化(MECE分类) → 金字塔(结论先行) |

## 提醒

- 以上模型起源于特定的方法论体系，使用时注意场景边界，不要机械套用
- AI可以充当"结构化助手""逻辑审计师""提问教练"等角色，善用AI加速思考
- 所有模型的最终目的不是"套模板"，而是**让思考从混沌走向清晰**
